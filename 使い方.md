# Bananacraft 使い方ガイド（日本語）

[Bananacraft](https://github.com/flll/Bananacraft) は、**Google Gemini** で街のコンセプトや建物デザインを考え、**Minecraft サーバー**上にブロック配置やボットで反映する **AI 都市開発**向けのツールです。画面は **Streamlit**（ブラウザ）、建設ボットは **Node.js（Mineflayer）** です。

---

## このリポジトリに含まれるもの

| パス | 役割 |
|------|------|
| `app/main.py` | メインの Streamlit アプリ（フェーズ 0〜3 の UI） |
| `app/` | 設計・ボクセル化・RCON などの Python モジュール |
| `AI_Carpenter_Bot/` | Mineflayer 製の「AI Carpenter」ボット |
| `projects/` | プロジェクトごとの生成物（画像・JSON など）が保存される |
| `setup.sh` | Ubuntu 向けの依存関係一括セットアップ（`sudo apt` 使用） |
| `start.sh` | 仮想環境を有効化して Streamlit を起動 |
| `deployment/` | systemd 用ユニットファイル（本番サーバー向け） |
| `.env.example` | 環境変数のテンプレート |

---

## 前提条件

1. **Python 3**（venv が使えること）
2. **Node.js**（AI Carpenter を使う場合。`AI_Carpenter_Bot` は npm 依存）
3. **Minecraft Java Edition サーバー**（ローカルまたはリモート）
   - **Docker（推奨）**: Purpur **26.1.2**・フラット・EssentialsX / WorldEdit。`make mc-reset`（初回）→ `make mc-up`
   - ゲーム: **`localhost:28888`**（`.env` の `MC_PORT`）
   - **RCON**: **`localhost:28889`**（`.env` の `RCON_PORT` / `RCON_PASSWORD`）

### Minecraft サーバー側

**Docker**: [minecraft/README.md](minecraft/README.md)。`make mc-attach` でコンソール接続（デタッチ: Ctrl+P → Ctrl+Q）。

**手動サーバー**（`server.properties` の例）:

- `enable-rcon=true`
- `rcon.port` / ゲームポートを `.env` の `RCON_PORT` / `MC_PORT` に合わせる

オンラインモード・ホワイトリスト・ファイアウォールは自分の環境に合わせて調整してください。

## セットアップ

### 方法 A: Ubuntu / Debian で `setup.sh` を使う（GCE 想定）

リポジトリのルートで:

```bash
cd /path/to/Bananacraft
chmod +x setup.sh
./setup.sh
```

`sudo` でパッケージと Node 22 を入れます。完了後、環境変数を設定します（次の節）。

### 方法 B: 手動（どの OS でも近い手順）

```bash
cd /path/to/Bananacraft
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cd AI_Carpenter_Bot && npm install && cd ..
mkdir -p projects
```

---

## 環境変数（`.env`）

```bash
cp .env.example .env
```

編集する主な項目:

| 変数 | 説明 |
|------|------|
| `GEMINI_API_KEY` | Gemini API キー（画像・テキスト生成に使用） |
| `RCON_HOST` | RCON の接続先（`make run` 時は `localhost`。`make stack-up` の Bananacraft コンテナ内は自動で `minecraft`） |
| `RCON_PORT` | RCON ポート（Docker MC 既定: **28889**） |
| `RCON_PASSWORD` | RCON パスワード（Docker MC と一致） |
| `MC_HOST` / `MC_PORT` | ボット接続先（既定: `localhost` / **28888**） |
| `MC_MEMORY` 等 | Docker MC のメモリ等（バージョンは Compose で Purpur 26.1.2 固定） |
| `STREAMLIT_PASSWORD` | 空ならログインなし。設定するとアクセスにパスワードが必要 |

## アプリの起動

リポジトリの**ルートディレクトリ**で:

```bash
make install    # 初回のみ
make mc-reset   # 初回 or サーバー種別変更時（確認あり）
make mc-up      # Purpur MC 起動（初回は数分）
make run        # Streamlit（http://127.0.0.1:8501）
```

従来どおり `start.sh` や `streamlit run app/main.py` でも可。UI も Docker に載せる場合は `make stack-up`。

ブラウザで **`http://localhost:8501`** を開きます（ポートは Streamlit の表示に従ってください）。

**サイドバー**にも Gemini API Key の入力欄があります。`.env` に書いても、ここで入力しても動作します（セッション中は入力値が優先されます）。

## 画面の流れ（フェーズ）

アプリは大まかに次の段階に進みます。

### Phase 0: プロジェクト作成

- プロジェクト名（半角英数推奨）を入力して作成
- 既存プロジェクトなら `projects/<名前>/` に保存されたデータを読み込みます
- **Gemini API Key が未設定だとエラー**になります

### Phase 1: コンセプトと区画（ゾーニング）

- 街のイメージを文章で入力 → **コンセプト生成**でプロンプト整理とコンセプトアート画像
- フィードバックで修正も可能
- 区画プランの生成・調整（ゾーンごとの建物候補など）

### Phase 2: 建物デザイン（ゾーンを 1 つ選ぶ）

- 「Design This」で対象ゾーンを選択
- デザイン画像の生成（コンセプト / 構造）
- **Create Blueprint** で構造解析 → 建築指示 JSON
- **Generate Construction Data** でブロック配置データ
- 3D プレビュー確認後、**Proceed to Site (Phase 3)** で現場フェーズへ

### Phase 3: 建設・連携

- **World Settings（サイドバー）**で建設原点（Origin X/Y/Z）を設定し **Save Origin**  
  - マイクラ上で建てたい「南西の基準点」に合わせます
- 構造の RCON 配置、装飾プラン生成、**Run AI Carpenter (Auto)** でボットによる自動実行 など

README にあるとおり、**装飾をボットで実行する**操作は Phase 3 の **「Run AI Carpenter (Auto)」** から行います。

## AI Carpenter ボット（コマンドライン）

UI 以外に、手動でボットを動かすこともできます。

```bash
cd AI_Carpenter_Bot
node index.js <プロジェクト名> [OriginX OriginY OriginZ] [ファイル名]
```

- 第 1 引数: `projects/<プロジェクト名>/` 以下のプロジェクト名
- 座標省略時: 近くのプレイヤーに合わせて原点を同期しようとします
- 最後の引数省略時: `decoration.json` を読みます（別ファイルならファイル名を指定）

**注意:** ボットは `.env` の **`MC_HOST` / `MC_PORT`**（既定 `localhost:28888`）に接続します。

## 本番運用（systemd）

`deployment/bananacraft.service` と `deployment/minecraft.service` は例です。**ユーザー名とパス**（`User=`、`WorkingDirectory=`、`ExecStart=`）を自分のサーバーに合わせて編集してから:

```bash
sudo cp deployment/bananacraft.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now bananacraft
```

Minecraft 本体の jar 配置や JVM オプションは各環境で別途管理してください。

## トラブルシューティング（よくあること）

| 現象 | 確認すること |
|------|----------------|
| API Key エラー | `.env` またはサイドバーのキー入力 |
| RCON で設置できない | `enable-rcon`、ポート、パスワードの一致、ファイアウォール |
| ボットが入れない | `make mc-up` 済みか、`.env` の `MC_PORT=28888`、`online-mode`、ホワイトリスト |
| `ModuleNotFoundError` | リポジトリ**ルート**から `streamlit run app/main.py`、venv を有効化しているか |
| プロジェクトが空 | `projects/` が存在するか、プロジェクト名の typo |
